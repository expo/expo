In this chapter, let's learn about the fundamentals of Expo Router library to implement stack and bottom tabs navigators with two routes.

## Expo Router basics

Expo Router is a routing framework for React Native and web applications. It allows us to manage navigation between different screens by using the same components across multiple platforms. To get started, we need to know about the following fundamentals and conventions:

- **app directory**: It is a special directory. Any files added to this directory becomes a screen inside our native app and reflects the same for the page on web.
- **Root layout**: The **app/\_layout.tsx** file. It used to define shared UI elements such as headers and tab bars so they persist consistently between different routes.
- **File name conventions**: Files name **index.tsx** match the parent directory and do not add a path segment. For example, the **index.tsx** file in **app** directory matches `/` route.
- A **route** file is defined by exporting a React component as its default value. It can use either `.js`, `.jsx`, `.ts`, or `.tsx` extension.
- Android, iOS, and web share a unified navigation structure.

> The above list is enough for us to get started. For a complete list of features, see [Introduction to Expo Router](/router/introduction/).

## Add a new screen to the stack

Let's create a new file called **about.tsx** inside the **app** directory. It displays the screen name when the user navigates to `/about` route.

```tsx app/about.tsx|collapseHeight=300
import { Text, View, StyleSheet } from 'react-native';

export default function AboutScreen() {
  return (
    <View style={styles.container}>
      <Text style={styles.text}>About screen</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#25292e',
    justifyContent: 'center',
    alignItems: 'center',
  },
  text: {
    color: '#fff',
  },
});
```

Inside **\_layout.tsx**:

1. Add a `<Stack.Screen />` component and an `options` prop to update the title of `/about` route.
2. Update `/index` route's title to `Home` by adding `options` prop.

{/* prettier-ignore */}
```tsx app/_layout.tsx
import { Stack } from 'expo-router';

export default function RootLayout() {
  return (
    <Stack>
      /* @info Add <CODE>options</CODE> prop to <CODE>/index</CODE> route. Add another <CODE>Stack.Screen</CODE> component to add <CODE>/about</CODE> route. */
      <Stack.Screen name="index" options={{ title: 'Home' }} />
      <Stack.Screen name="about" options={{ title: 'About' }} />
      /* @end */
    </Stack>
  );
}
```

<Collapsible summary={<>What is a <CODE>Stack</CODE>?</>}>

A stack navigator is the foundational way of navigating between different screens in an app. On Android, a stacked route animates on top of the current screen. On iOS, a stacked route animates from the right. Expo Router provides a `Stack` navigation component that creates a navigation stack and allows you to add new routes in your app.

## Navigate between screens

To navigate from `/index` route to `/about` route, we will use Expo Router's built-in component called `Link`. It is a React component that renders a `<Text>` with a given `href` prop.

1. Import `Link` component from `expo-router` inside **index.tsx**.
2. Add a `Link` component after the `<Text>` component and pass the `href` prop with `/about` route.
3. Add a style of `fontSize`, `textDecorationLine`, and `color` to the `Link` component. It takes the same props as `<Text>` component.

{/* prettier-ignore */}
```tsx app/index.tsx|collapseHeight=300
import { Text, View, StyleSheet } from 'react-native';
/* @info Import <CODE>Link</CODE> component from <CODE>expo-router</CODE>. */ import { Link } from 'expo-router'; /* @end */

export default function Index() {
  return (
    <View style={styles.container}>
      <Text style={styles.text}>Home screen</Text>
      /* @info Add <CODE>Link</CODE> component after <CODE>Text</CODE> component and pass the <CODE>href</CODE> prop with <CODE>/about</CODE> route. */
      <Link href="/about" style={styles.button}>
        Go to About screen
      </Link>
      /* @end */
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#25292e',
    alignItems: 'center',
    justifyContent: 'center',
  },
  text: {
    color: '#fff',
  },
  /* @info Add the style of <CODE>fontSize</CODE>, <CODE>textDecorationLine</CODE>, and <CODE>color</CODE> to <CODE>Link</CODE> component. */
  button: {
    fontSize: 20,
    textDecorationLine: 'underline',
    color: '#fff',
  },
  /* @end */
});
```

## Add a not-found route

When a route doesn't exist, we can use a `+not-found` route to display a fallback screen. This is useful when we want to display a custom screen when a user navigates to a 404 route. Expo Router uses a special **+not-found.tsx** file to handle this case.

1. Let's create a new file called **+not-found.tsx** inside **app** directory and add the `NotFoundScreen` component.
2. Use `options` prop from `Stack.Screen` component to display a custom screen title for `+not-found` route.
3. Add a `Link` component to navigate to `/` route which is our fallback route.

{/* prettier-ignore */}
```tsx app/+not-found.tsx
import { View, StyleSheet } from 'react-native';
import { Link, Stack } from 'expo-router';

export default function NotFoundScreen() {
  return (
    <>
      /* @info Add <CODE>Stack.Screen</CODE> component with <CODE>options</CODE> prop to update the title of <CODE>+not-found</CODE> route. */
      <Stack.Screen options={{ title: 'Oops! Not Found' }} />
      /* @end */
      <View style={styles.container}>
        /* @info Adding a fallback route allows the user to navigate to that screen. Here, the fallback route is Home screen. */
        <Link href="/" style={styles.button}>
          Go back to Home screen!
        </Link>
        /* @end */
      </View>
    </>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#25292e',
    justifyContent: 'center',
    alignItems: 'center',
  },

  button: {
    fontSize: 20,
    textDecorationLine: 'underline',
    color: '#fff',
  },
});
```

4. Update the `Stack` in **app/\_layout.tsx** file to add `+not-found` route.

{/* prettier-ignore */}
```tsx app/_layout.tsx
import { Stack } from 'expo-router';

export default function RootLayout() {
  return (
    <Stack>
      <Stack.Screen name="index" options={{ title: 'Home' }} />
      <Stack.Screen name="about" options={{ title: 'About' }} />
      /* @info Adding <CODE>+not-found</CODE> route in <CODE>RootLayout</CODE> will display this fallback screen when a user navigates to a 404 route. */
      <Stack.Screen name="+not-found" />
      /* @end */
    </Stack>
  );
}
```

## Add a bottom tab navigator

At this point, the file structure of our **app** directory looks like the following:

<FileTree
  files={[
    ['app/_layout.tsx', 'Root layout'],
    ['app/index.tsx', "matches route '/'"],
    ['app/about.tsx', "matches route '/about'"],
    ['app/+not-found.tsx', 'matches route any 404 route'],
  ]}
/>

We will add a bottom tab navigator to our app and re-use the existing Home and About screens. The purpose here is to create a tab layout (which is common navigation pattern in many apps, especially, social media apps like X or BlueSky) and use the stack navigator from the root layout so that the `+not-found` route is displayed over any other nested stack.

1. Inside **app** directory, add a **(tabs)** subdirectory. The `(tabs)` is a special directory that allows us to group routes together and display them in a bottom tab bar.
2. Create a **(tabs)/\_layout.tsx** file inside the directory. This file will be used to define the tab layout which is separate Root layout.
3. Move the existing **index.tsx** and **about.tsx** files inside **(tabs)** directory. The structure of **app** directory should look like this now:

<FileTree
  files={[
    ['app/_layout.tsx', 'Root layout'],
    ['app/+not-found.tsx', 'matches route any 404 route'],
    ['app/(tabs)/_layout.tsx', 'Tab layout'],
    ['app/(tabs)/index.tsx', "matches route '/'"],
    ['app/(tabs)/about.tsx', "matches route '/about'"],
  ]}
/>

4. Update the Root layout (**app/\_layout.tsx**) file to add a `Tabs` component and a `(tabs)` route.

{/* prettier-ignore */}
```tsx app/_layout.tsx
import { Stack } from 'expo-router';

export default function RootLayout() {
  return (
    <Stack>
      /* @info This is how a tab navigator is nested inside a stack navigator, especially when the Root layout is composed of a parent stack navigator. We're also setting the <CODE>headerShown</CODE> option to <CODE>false</CODE> to hide the header for the tab navigator. Otherwise, there will be two headers displayed on each screen. */
      <Stack.Screen name="(tabs)" options={{ headerShown: false }} />
      /* @end */
      <Stack.Screen name="+not-found" />
    </Stack>
  );
}
```

5. Create a **(tabs)/\_layout.tsx** file and use `Tabs` component to define the tab layout.

{/* prettier-ignore */}
```tsx app/(tabs)/_layout.tsx
import { Tabs } from 'expo-router';

export default function TabLayout() {
  return (
    <Tabs>
      /* @info <CODE>Tabs.Screen</CODE> component works in a similar way and accepts same props as <CODE>Stack.Screen</CODE> component. The only difference is the navigation pattern on the device. */
      <Tabs.Screen name="index" options={{ title: 'Home' }} />
      <Tabs.Screen name="about" options={{ title: 'About' }} />
      /* @end */
    </Tabs>
  );
}
```

## Update bottom tab navigator appearance

Right now, the bottom tab navigator looks the same on all platforms but doesn't really matches style of our app. For example, there are no custom icons displayed on the tab bar or header and bottom tab background color doesn't match the background color of the app.

Modify the **(tabs)/\_layout.tsx** file to add tab bar icons:

1. Import `Ioncicons` icons set from [`@expo/vector-icons`](/guides/icons/#expovector-icons) &mdash; a library that includes popular icon sets.
2. Add `tabBarIcon` option to both `index` and `about` routes. It is a function that takes `focused` and `color` as params, and renders the icon component. From the icon set, we can provide custom icon names.
3. Add `screenOptions.tabBarActiveTintColor` to `Tabs` component and set its value to `#ffd33d`. This will change the color of the tab bar icon and label when active.

{/* prettier-ignore */}
```tsx app/(tabs)/_layout.tsx
import { Tabs } from 'expo-router';

/* @info Import <CODEIonicons</CODE> icon set.*/

import Ionicons from '@expo/vector-icons/Ionicons';

/* @end */

export default function TabLayout() {
  return (
    <Tabs
      /* @info There are many <CODE>screenOptions</CODE> we can use to customize the tab bar. We're changing the active tab color here to custom value which we will also use later in our app. */
      screenOptions={{
        tabBarActiveTintColor: '#ffd33d',
      }}
      /* @end */
    >
      <Tabs.Screen
        name="index"
        options={{
          title: 'Home',
          tabBarIcon: ({ color, focused }) => (
            /* @info The <CODE>focused</CODE> param allows us to change a tab's icon and label behavior when it is active and inactive.*/
            <Ionicons name={focused ? 'home-sharp' : 'home-outline'} color={color} size={24} />
            /* @end */
          ),
        }}
      />

      <Tabs.Screen
        name="about"
        options={{
          title: 'About',
          tabBarIcon: ({ color, focused }) => (
            <Ionicons
              name={focused ? 'information-circle' : 'information-circle-outline'}
              color={color}
              size={24}
            />
          ),
        }}
      />
    </Tabs>
  );
}
```

Let's also change the background color of the tab bar and header using `screenOptions` prop.

```tsx app/(tabs)/_layout.tsx
<Tabs
  screenOptions={{
    tabBarActiveTintColor: '#ffd33d',
    headerStyle: {
      backgroundColor: '#25292e',
    },
    headerShadowVisible: false,
    headerTintColor: '#fff',
    tabBarStyle: {
    backgroundColor: '#25292e',
    },
  }}
>
```

- The background of the header is set to `#25292e` using `headerStyle` property. We have also also disabled the shadow of the header using `headerShadowVisible`.
- The `headerTintColor` applies `#fff` to the header label.
- The `tabBarStyle.backgroundColor` applies `#25292e` to the tab bar.
